# Concerns

## Tech Debt
- **Intentional type duplication** -- `IKadenaKeypair` is canonically defined in `src/signing/types.ts:22-30` but a structurally identical copy still exists in `src/interactions/ouroFunctions.ts` for Phase 2b backward-compat; consumers like `src/interactions/wrapFunctions.ts:18` import it from there. `CLAUDE.md:48-50` flags this as deliberate -- do NOT consolidate without checking the comment trail
- **Phase-1 temporary copies** -- `src/signing/primitives.ts:1-14` openly states the file is a TEMP duplicate of OuronetUI's `src/lib/universalSign.ts`, kept byte-identical until Phase 3 fully consolidates. `src/signing/universalSign.ts:1-9` carries the same warning
- **Scaffolded-but-unused interfaces** -- `KeyResolver`, `PactClient`, `SigningStrategy` were introduced as a public-API scaffold ahead of internal usage (`src/signing/types.ts:6-9`); the comment notes "nothing inside core USES them yet"
- **Deprecated aliases retained** -- `GAS_STATION` and `NATIVE_TOKEN_VAULT` in `src/constants/kadena.ts:47-50` are `@deprecated` JSDoc-flagged but still exported (used by `src/interactions/wrapFunctions.ts:13-15` among others); cannot be removed without a major version bump
- **Static `KADENA_BASE_URL` pinned to node2** -- `src/constants/kadena.ts:17` builds a baked URL using the primary host literal rather than the failover-aware `getActiveHost()`; the dynamic `getPactUrl(chainId)` exists alongside it and is the recommended path, leaving the static const as legacy
- **No `TODO`/`FIXME`/`HACK` markers in source** -- only the two `@deprecated` aliases above; tech-debt is documented in JSDoc paragraphs rather than inline tags

## Known Issues
- **Heavy use of `: any`** -- ~169 occurrences across 20 files, concentrated in `src/interactions/dexFunctions.ts` (44), `src/interactions/addLiquidityFunctions.ts` (21), `src/interactions/ouroFunctions.ts` (38), reflecting un-typed Pact response shapes. `AccountSelectorData["ouronet-account-guard"]: any` and `governor: any | false` are explicitly documented as polymorphic Pact guards (`src/interactions/ouroFunctions.ts:19-26`, `:57`)
- **Empty `} catch {` blocks** -- 13+ occurrences swallow errors silently: `src/network/nodeFailover.ts:43` (intentional, returns `false` for health check), `src/crypto/v2.ts:127,183` (recovery paths), `src/interactions/dexFunctions.ts:1318,1347,1512,1523,1533`, `src/interactions/addLiquidityFunctions.ts:256,274,939`. Most are recovery fallbacks but several discard the actual error context that would aid debugging
- **`console.*` calls in library code** -- ~103 occurrences across 15 files (`src/interactions/infoOneFunctions.ts`: 23, `src/interactions/ouroFunctions.ts`: 18, `src/interactions/dexFunctions.ts`: 27); a published library writing to consumer's stdout/stderr without an injection point. No logger abstraction is offered
- **Codex format frozen at `"1.2"`** -- the version string MUST NOT change because every existing user's downloaded `OuronetCodex_*.json` carries it; bumping breaks recovery. Future format changes require a separate `CodexExportV2` type rather than mutating the V1.2 path (`src/codex/codec.ts:8-13`)
- **`createDefaultRegistry()` registers Genesis only** -- `Leto`, `Artemis`, `Apollo`, `createGen1Primitive` are exported but NOT in the default registry; consumers who silently expect them in the default set will get `undefined` from `registry.get(id)` (`src/dalos/index.ts:77-91`)
- **Smart-account branch selection is consumer-side** -- `CodexSigningStrategy.execute` still takes a single AND-of-keysets `guards` array; the UI must analyze the three branches and pick one before calling execute. A consumer that forgets this for a Σ. account will sign against the wrong branch (`src/guard/smartAccountAuth.ts:46-50`)
- **`}\\ catch { return ... }` short-circuits in dex/addLiquidity** swallow exceptions and return sentinels like `{ valid: false }`, `[]`, `"N/A"` (`src/interactions/addLiquidityFunctions.ts:939`, `src/interactions/dexFunctions.ts:1523,1533`); these mask real failures from upstream calls

## Performance
- **Default Pact reader is uncached** -- `rawCalibratedDirtyRead` runs each request against the chain with no dedup. OuronetUI mitigates by calling `setPactReader(calibratedDirtyRead)` at boot; a consumer that forgets the wiring will see per-keystroke flicker on Smart Swap-style UIs (the very issue the pluggable seam was introduced to fix, per `src/reads/pactReader.ts:6-13`)
- **Read simulation gas ceiling is 10M** -- intentionally generous since simulation gas is "free" but means a malicious read could take longer to time out (`src/reads/rawCalibratedRead.ts:19-20`)
- **Node-failover retry interval 30s** -- once on fallback, primary recheck happens every 30s (`src/network/nodeFailover.ts:30`); a flaky primary will keep tripping the swap during the retry window
- **No request batching surface** -- each `pactRead`/`createClient(...).submit` call is independent; no bulk-read API exposed by core (consumers must wrap)

## Security
- **Hardcoded node hosts** -- `https://node2.stoachain.com`, `https://node1.stoachain.com` baked into `src/network/nodeFailover.ts:13-14` and `src/constants/kadena.ts:17`. Custom node configuration exists via `setNodeConfig("custom", customUrl, ...)` but the defaults are pinned to two domains controlled by the project
- **Health check has no auth/integrity verification** -- `isHealthy(host)` only checks `res.ok` on `${host}/info` (`src/network/nodeFailover.ts:36-46`); does not verify response signature, network id, or chainweb version, so a hostile fallback URL configured via `setNodeConfig("custom", url)` would be trusted blindly
- **Encrypted secret material flows through `IKadenaKeypair.encryptedSecretKey: unknown`** -- typed as `unknown` to avoid binding to `@kadena/hd-wallet`'s internals (`src/signing/types.ts:27`); consumers must hold the password in-process for the chainweaver/ecko WASM signing path (`src/signing/universalSign.ts:32-44`)
- **`@deprecated` constants can mask gas-station impersonation** -- `GAS_STATION` and `NATIVE_TOKEN_VAULT` aliases (`src/constants/kadena.ts:48-50`) point at protocol-level `c:` accounts; consumers that hardcode the literal string elsewhere instead of importing get no compile-time check that they're using the canonical source
- **No input validation on `setPactReader` / `setNodeConfig`** -- both accept any function/string at boot and silently overwrite the active state for the rest of the process lifetime; a misconfigured boot order in a downstream consumer can route reads through an attacker-controlled callback (`src/reads/pactReader.ts:52-54`, `src/network/nodeFailover.ts:128-151`)
- **`v: 2` envelope uses PBKDF2-SHA512 600k iterations + AES-GCM-256** -- modern parameters but encryption upgrade from V1 (`smartDecrypt` / `isCodexUpgraded`) keeps a V1 read-path open until upgrade completes (`src/crypto/v2.ts:1-17`); a downgrade attack on a stored blob is mitigated only by the `v: 2` marker in the JSON envelope
- **License is `UNLICENSED`** despite the package being published as `--access public` to npmjs.org (`package.json:115`, `package.json:113`); consumers technically have no granted rights to the source on npm
