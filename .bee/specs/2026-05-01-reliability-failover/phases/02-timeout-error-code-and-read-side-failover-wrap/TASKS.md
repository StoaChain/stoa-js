## Goal

Introduce the `TIMEOUT` error classification helper next to the existing `createSigningError` / `createSimulationError` factories, then wrap the uncached reader's `dirtyRead` invocation with `withFailover` plus a 15 s timeout using inline `Promise.race` + `AbortController` defence-in-depth. Because every read in the interactions layer routes through `pactRead` → `rawCalibratedDirtyRead` (post-v2.0.0), this single edit propagates failover and timeout coverage to all 16 already-migrated read sites without touching any of them. Extend the `pactRead` and `rawCalibratedDirtyRead` options bags with a `readTimeoutMs?: number` pass-through so per-call overrides are possible (default 15000 ms when omitted).

This phase deliberately uses an inline `Promise.race` + `AbortController` block (NOT a shared helper) — Phase 3 introduces `runWithTimeout(...)` as part of the `getFailoverClient` factory and refactors this inline block to consume it, eliminating the temporary duplication. The deferral is acknowledged-and-locked per the atomic-ship contract from Phase 1.

NO new tests in Phase 2. Test coverage for `TIMEOUT` classification and read-side failover lands in Phase 4 (REQ-14 + REQ-15). Phase 2's gate is "existing tests stay green" plus typecheck.
